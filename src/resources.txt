# Prompt replicable — Fraudia Frontend

Este prompt describe, paso a paso, el sistema que debe quedar construido. Un
agente que lo lea de principio a fin debe poder reproducir el frontend
exacto sobre un proyecto Vite + Vue 3 + TypeScript vacío.

---

## 1. Contexto

Construye el panel de control frontend de **Fraudia**, un sistema de detección
asistida de fraude en siniestros de seguros. El backend procesa siniestros con
un motor de reglas + ML y persiste los resultados en la tabla
`fraud_ia.siniestros_scored_final` (DDL en
`src/resources/create_siniestros_scored_final.sql`). La salida del pipeline es
**una alerta para revisión humana, nunca una acusación automática**; el copy
y el diseño deben reflejar ese tono.

Idioma de la UI: **español (es-CO)**. Modo: **dark only**. Estilo: minimalista
tipo Linear / Notion / Jira.

---

## 2. Stack obligatorio

- Vue 3 con `<script setup lang="ts">` (Composition API).
- TypeScript estricto (mantener `noUnusedLocals`, `noUnusedParameters`,
  `erasableSyntaxOnly`, `noFallthroughCasesInSwitch`).
- Vite 8 + `@vitejs/plugin-vue`.
- Tailwind CSS v4 vía `@tailwindcss/vite` (sin `tailwind.config.js`; el theme
  va en `src/style.css` con la directiva `@theme`).
- Sin Vue Router ni Pinia: navegación entre vistas con un `ref<ViewKey>` en
  `App.vue`. SPA de 3 secciones.
- Iconos: SVG inline en `src/components/icons/` (no librerías externas).

Comandos esperados:

```
npm install
npm run dev
npm run build      # vue-tsc -b && vite build
npm run preview
```

---

## 3. Layout global

`App.vue` debe contener:

- Una barra lateral fija (`Sidebar.vue`) con 3 ítems: **Dashboard**,
  **Subir archivos**, **Preguntas al chat**. En desktop (`lg`+) la sidebar es
  `sticky`; en mobile aparece como drawer con backdrop blur y se abre desde un
  botón hamburguesa en un header móvil.
- Un `<main>` que renderiza la vista activa con una transición `fade + translateY`
  corta (200 ms) entre vistas.
- Estado: `current: 'dashboard' | 'upload' | 'chat'` y `sidebarOpen: boolean`.

La sidebar muestra:
- Logo cuadrado redondeado (gradiente violeta → fucsia, letra "F").
- Marca "Fraudia" + subtítulo "Panel de control".
- Sección "Navegación" con 3 botones; el activo lleva una barra vertical
  violeta a la izquierda y fondo `--color-surface-3`.
- Tarjeta inferior con avatar circular y "Sesión activa".

  - Amarillo `#fbbf24` + bg `rgba(251,191,36,0.08)`

Globales:

- Animación utilitaria `fade-up` (`opacity 0 → 1`, `translateY 6px → 0`, 250 ms).
- Tipografía Inter / system stack, `tabular-nums` en columnas numéricas.

## 5. Modelo de datos

Crear `src/types/siniestro.ts` que refleje **uno a uno** la tabla
`fraud_ia.siniestros_scored_final`. Mínimo a tipar:

- `SemaforoFinal = 'Rojo' | 'Amarillo' | 'Verde'`.
- `ReglaCritica` con las 12 banderas (`rf_01_perdida_total_robo` …
  `rf_12_alta_frecuencia_y_borde_vigencia`).
- `AlertaScore` con las 14 alertas `score_*` (sin `score_total_reglas`).
- `Siniestro` con todos los campos de la SQL: IDs, fechas (`string | null`,
  formato ISO `YYYY-MM-DD`), montos (`number | null`), strings, booleanos,
  frecuencias, similitudes, IDs similares (`number[]`), los 14 sub-scores
  enteros, `score_total_reglas`, los 4 semáforos, `probabilidad_ml`,
  `prediccion_ml: 0 | 1 | null`, `score_final`, `reglas_criticas_activadas`,
  `alertas_score_activadas`, `explicabilidad`.
- Diccionario `REGLA_LABELS: Record<ReglaCritica, string>` con etiquetas
  legibles para mostrar en UI.

Si la SQL cambia, este archivo se actualiza **primero**.

## 7. Mock de datos

`src/data/dashboard.json` debe ser un **array** de `Siniestro` con al menos 7
registros que cubran los 3 semáforos (≈3 Rojo, ≈2 Amarillo, ≈2 Verde). Cada
registro debe respetar las constraints de la SQL: `score_final` 0–100,
`probabilidad_ml` 0–1, `prediccion_ml` 0|1|null, semáforos en
`{Rojo, Amarillo, Verde}`, arrays JSON válidos. Usa montos en COP, ramos
realistas (Autos, Hogar) y coberturas que disparen reglas (`rf_01`, `rf_05`,
`rf_07`, etc.).

## 8. Vista Dashboard

`src/components/views/DashboardView.vue`:

- En `onMounted` llama `fetchSiniestros()` y mantiene `loading`, `error`,
  `siniestros: Siniestro[]`. Botón "Actualizar" con icono que rota durante el
  fetch.
- Header: título "Dashboard", subtítulo con `<code>semaforo_final</code>`.
- Banda de **4 KPIs** en grid (`grid-cols-2 sm:grid-cols-4`):
  1. Total siniestros.
  2. Monto reclamado total (formato corto: `$X k / $X M / $X mM`).
  3. Score promedio (1 decimal).
  4. Críticos (Rojo) en color rojo suave.
- Tres columnas kanban (`md:grid-cols-3`) — Rojo / Amarillo / Verde — cada una
  con:
  - Header fijo: punto de color con `ring`, label, descripción contextual
    ("Riesgo crítico · revisión humana", "Riesgo moderado · seguimiento",
    "Riesgo bajo · flujo normal") y badge contador a la derecha.
  - Cuerpo scrollable vertical.
  - Si `loading`: 3 skeletons `animate-pulse` por columna.
  - Si vacío: mensaje "Sin siniestros en esta categoría".
  - Si hay datos: lista de `<SiniestroCard>` con `fade-up` escalonado
    (`animationDelay = i * 40ms`).
- Si `error`: banner rojo con el mensaje.

`src/components/views/SiniestroCard.vue` (recibe `siniestro: Siniestro`):

- Header: punto del color de su semáforo + `#id_siniestro` en mono pequeño;
  título = `cobertura`; subtítulo = `ramo · sucursal`. A la derecha, badge con
  `score_final` (1 decimal) coloreado según semáforo + label "score" debajo.
- Mini-grid de 2 columnas:
  - "Reclamado" → `Intl.NumberFormat('es-CO', { style: 'currency', currency: 'COP', maximumFractionDigits: 0 })`.
  - "Prob. ML" → `Math.round(probabilidad_ml * 100) + '%'`, en el color del semáforo.
- `explicabilidad` truncada a 2 líneas (`line-clamp-2`).
- Footer: fecha de ocurrencia (icono calendario) y badges discretos:
  - Conteo de `reglas_criticas_activadas` con icono triángulo.
  - "docs" si `docs_faltantes || docs_inconsistentes`.
  - `×N` si `freq_asegurado_18m >= 3`.

Tarjetas: `rounded-xl`, borde `--color-border`, fondo `--color-surface-2`,
hover `-translate-y-px` + sombra `shadow-black/30`.

## 9. Vista Subir archivos

`UploadView.vue`:

- Header: título "Subir archivos", subtítulo "Selecciona o arrastra un archivo
  `.csv` para procesarlo".
- Zona drag & drop centrada (`max-w-2xl`):
  - Borde discontinuo, esquina redondeada `rounded-2xl`.
  - Icono upload en círculo redondeado (cambia a violeta en hover).
  - Texto "Arrastra tu archivo aquí o **haz clic para seleccionar**" + nota
    "Solo archivos CSV · Máximo 10MB".
  - `tabindex="0"`, abre el `<input type="file" accept=".csv,text/csv">` con
    Enter / Espacio / clic.
  - Estado dragging: borde violeta + fondo tenue + scale `1.01`.
- Validación: extensión `.csv` o MIME `text/csv` / `application/vnd.ms-excel`.
  Si no es CSV → `alert('Solo se permiten archivos .csv')`.
- En éxito → `alert('Archivo subido correctamente')` y mostrar tarjeta de
  confirmación con check verde, nombre del archivo y botón "Quitar".
- **Sin backend**: nada de `fetch` aquí.

## 10. Vista Preguntas al chat

`ChatView.vue`:

- Header: título "Preguntas al chat", subtítulo "Conversa con el asistente
  para resolver tus dudas".
- Estado `messages: { id, role: 'user' | 'bot', text }[]`. Inicia con un
  mensaje del bot:
  > "Hola, soy tu asistente virtual. ¿En qué puedo ayudarte?"
- Lista de mensajes en contenedor `max-w-3xl` centrado:
  - Burbuja del **usuario** a la derecha, fondo `--color-accent`, texto blanco,
    `rounded-2xl rounded-br-md`, avatar "U" a la derecha.
  - Burbuja del **bot** a la izquierda, fondo `--color-surface-2`, borde sutil,
    `rounded-2xl rounded-bl-md`, avatar "F" con gradiente violeta → fucsia.
- Mientras el bot "responde": indicador con 3 puntos animados
  (`animation-delay` escalonado a 0/150/300 ms).
- Scroll automático al final con `nextTick` + `scrollTo({ behavior: 'smooth' })`
  cada vez que llega un mensaje.
- Input + botón enviar (icono avión de papel). Envía con `Enter` (sin Shift).
  Botón deshabilitado si el input está vacío.
- Respuesta del bot **hardcodeada**: tras 800 ms responde otra vez:
  > "Hola, soy tu asistente virtual. ¿En qué puedo ayudarte?"

## 12. Criterios de aceptación

1. `npm run build` corre `vue-tsc -b && vite build` sin warnings ni errores.
2. No hay errores de lint en `src/`.
3. La sidebar es funcional en mobile (drawer + backdrop) y desktop (sticky).
4. El Dashboard:
   - Muestra skeletons al cargar.
   - Llena las 3 columnas con los datos del mock al terminar.
   - El botón "Actualizar" rehidrata los datos.
   - KPIs reflejan los totales del array.
5. Subir archivos:
   - Acepta `.csv` por click y por drop.
   - Rechaza otros formatos con `alert`.
   - En éxito muestra `alert` y tarjeta de confirmación.
6. Chat:
   - Mensaje inicial del bot visible.
   - Enviar mensaje del usuario muestra typing indicator y luego la respuesta
     hardcodeada.
   - Scroll va al final automáticamente.
7. Conectar al backend real solo requiere cambiar el cuerpo de
   `fetchSiniestros()` en `src/services/siniestros.ts`.

## 13. Lo que NO se debe hacer

- No introducir Vue Router, Pinia, axios, librerías de iconos ni CSS frameworks
  adicionales.
- No agregar `tailwind.config.js`; v4 usa `@theme` en CSS.
- No mezclar light mode.
- No agregar comentarios narrativos al código.
- No crear documentación adicional salvo que se pida explícitamente.
- No usar copy acusatorio: el sistema **alerta**, no condena.
