# Endpoints — Fraudia Frontend

Toda la comunicación con el backend está aislada en `src/services/*.ts`. Hoy
los servicios funcionan con datos simulados (`setTimeout` + mock) y la llamada
real `fetch()` está **comentada** en cada archivo. Para "encender" un endpoint
basta con descomentar y borrar la simulación; las vistas no necesitan cambios.

Patrón general:

- Cada vista importa una función pública de su servicio (`fetchSiniestros`,
  `uploadCsv`, `sendChatMessage`).
- El servicio devuelve tipos definidos en el mismo archivo o en
  `src/types/`.
- Errores siempre se lanzan con `throw new Error(...)` para que la vista los
  capture en su `try/catch` y los muestre.

---

## 1. Listar siniestros (Dashboard)

- Archivo: `src/services/siniestros.ts`
- Función: `fetchSiniestros(): Promise<Siniestro[]>`
- Consumido por: `src/components/views/DashboardView.vue` (en `onMounted` y al
  pulsar "Actualizar").
- Mock actual: `src/data/dashboard.json` con `setTimeout(350)`.

Endpoint esperado:

- Método: `GET`
- Ruta sugerida: `/api/siniestros`
- Body: —
- Respuesta: `Siniestro[]` con el shape de `src/types/siniestro.ts` (refleja
  `fraud_ia.siniestros_scored_final`).

Implementación real (descomentar dentro de la función):

```ts
const res = await fetch('/api/siniestros')
if (!res.ok) throw new Error(`HTTP ${res.status}`)
return (await res.json()) as Siniestro[]
```

Si el backend devuelve un wrapper (ej. `{ data: [...], meta: {...} }`),
mapearlo aquí y exponer solo el array a la vista.

## 2. Subida de CSV (Subir archivos)

- Archivo: `src/services/upload.ts`
- Función: `uploadCsv(file: File): Promise<UploadResult>`
- Consumido por: `src/components/views/UploadView.vue` (después de validar
  extensión `.csv`).
- Mock actual: resuelve `{ ok: true, fileName, size }` con `setTimeout(500)`.

Endpoint esperado:

- Método: `POST`
- Ruta sugerida: `/api/upload/csv`
- Body: `multipart/form-data` con el campo `file`.
- Respuesta sugerida: `{ ok: true, fileName: string, size: number }` (puede
  extenderse con `id`, `rows_processed`, etc.).

Implementación real (descomentar dentro de la función):

```ts
const form = new FormData()
form.append('file', file)
const res = await fetch('/api/upload/csv', {
  method: 'POST',
  body: form,
})
if (!res.ok) throw new Error(`HTTP ${res.status}`)
return (await res.json()) as UploadResult
```

Notas:

- **No** setear `Content-Type` manualmente: el navegador genera el `boundary`
  correcto automáticamente.
- Si el endpoint requiere autenticación, agregar
  `headers: { Authorization: ... }`.
- Para mostrar barra de progreso real usar `XMLHttpRequest` con
  `xhr.upload.onprogress` (`fetch` aún no expone progreso de subida).
- El frontend ya valida `.csv` por extensión y MIME (`text/csv`,
  `application/vnd.ms-excel`); el backend debe revalidar.

## 3. Chat (Preguntas al chat)

- Archivo: `src/services/chat.ts`
- Función: `sendChatMessage(message: string): Promise<ChatResponse>`
- Consumido por: `src/components/views/ChatView.vue` (al enviar un mensaje).
- Mock actual: responde con texto fijo (`FALLBACK_REPLY`) tras `setTimeout(800)`.

Endpoint esperado:

- Método: `POST`
- Ruta sugerida: `/api/chat`
- Body JSON: `{ "message": string }` (extender a `{ message, history }` si el
  backend no mantiene contexto).
- Respuesta sugerida: `{ reply: string }`.

Implementación real (descomentar dentro de la función):

```ts
const res = await fetch('/api/chat', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ message }),
})
if (!res.ok) throw new Error(`HTTP ${res.status}`)
return (await res.json()) as ChatResponse
```

Notas:

- Si el backend hace **streaming** (SSE / chunked), no usar `fetch().json()`;
  consumir el `ReadableStream` y actualizar el último mensaje del bot
  token a token. La función `send()` en `ChatView.vue` necesita reescribirse
  parcialmente para esto.
- Si requiere auth, agregar `headers: { Authorization: ... }`.
- Para mantener historial conversacional, pasar `messages.value` (filtrado a
  `{ role, text }`) en el body.

## 4. Variables de entorno (cuando aplique)

Hoy las rutas son relativas (`/api/...`) asumiendo proxy de Vite o
mismo origen. Si el backend vive en otro dominio:

1. Definir `VITE_API_URL` en `.env`.
2. Usar `${import.meta.env.VITE_API_URL}/siniestros` en cada `fetch`.
3. Considerar configurar CORS o un proxy en `vite.config.ts`:

```ts
server: {
  proxy: {
    '/api': 'http://localhost:8000',
  },
}
```

## 5. Resumen rápido

| Vista              | Servicio                        | Endpoint sugerido    | Método |
| ------------------ | ------------------------------- | -------------------- | ------ |
| Dashboard          | `src/services/siniestros.ts`    | `/api/siniestros`    | GET    |
| Subir archivos     | `src/services/upload.ts`        | `/api/upload/csv`    | POST   |
| Preguntas al chat  | `src/services/chat.ts`          | `/api/chat`          | POST   |

Para activar cualquiera: abrir el archivo del servicio, descomentar el
bloque marcado como "Cuando el backend esté listo" y eliminar el
`setTimeout` + retorno simulado.
